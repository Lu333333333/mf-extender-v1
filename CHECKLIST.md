# CHECKLIST.md — レビュー・リリースチェックリスト

PR のレビュアーとリリース担当者が使うチェックリスト。  
該当しない項目は `N/A` と記載してスキップ可。

---

## PR レビュー時

### 基本確認

- [ ] `manifest.json` の構文が正しい（`node -c` または JSON バリデータで確認）
- [ ] 変更した `content.js` の構文が正しい（`node -c <ファイル名>`）
- [ ] `manifest.json` の `version` が適切にインクリメントされている（リリース PR のみ）
- [ ] `.pem` ファイルがコミットに含まれていない
- [ ] `*.zip` `*.crx` がコミットに含まれていない

### React DOM 干渉チェック（最重要）

> `accounting.moneyforward.com` および `box.moneyforward.com` に関わる変更がある場合は必ず確認

- [ ] React が管理する要素（`tr` / `thead` / `td` 等）への `insertBefore` / `appendChild` が**ない**
- [ ] React が管理する要素への `innerHTML` の書き換えが**ない**
- [ ] 独自 UI の追加は `document.body` 直下への独立要素として実装されている
- [ ] 選択状態の表現に `classList.toggle` のみを使用している

### MutationObserver チェック

- [ ] `MutationObserver` のコールバックに `requestAnimationFrame` デバウンスが入っている
- [ ] `beforeunload` で `observer.disconnect()` が呼ばれている

### 機能別チェック（変更があった機能のみ）

**ストレージ一括DL（`storage/content.js`）**
- [ ] `#js-btn-bulk-destroy` の隣にボタンが追加される
- [ ] チェックなしの状態でボタンが `is-disabled` になる
- [ ] 複数選択 → 一括DL → ZIP が正常に開ける
- [ ] 並列数 `CHUNK_SIZE = 8` が維持されている

**クラウドBox一括DL（`box/content.js`）**
- [ ] `Ctrl + クリック` でトグル選択できる
- [ ] `Shift + クリック` で範囲選択できる（既存選択はクリア）
- [ ] `Esc` キーで全選択解除できる
- [ ] 修飾キーなしの通常クリックで詳細ページへ遷移する（既存動作が壊れていない）
- [ ] フローティングパネルが `document.body` 直下に設置されている
- [ ] DL URL が `/frontend/v3/files/{UUID}/download` パターンを使っている

**月次セレクトボックス（`journals/content.js`）**
- [ ] `recognized_at_from` / `recognized_at_to` の ID が実際の画面と一致している（DevTools で確認）
- [ ] 期首月がフィールドの初期値から正しく読み取れている
- [ ] セレクト選択後、開始日フィールドに `YYYY/MM/01` 形式で入力される
- [ ] セレクト選択後、終了日フィールドに `YYYY/MM/末日` 形式で入力される
- [ ] 開始月セレクトを選択した際、終了日フィールドにも自動で同月の末日が入力される
- [ ] 終了月セレクトで個別に月を選び直した場合、その値で終了日が上書きされる（開始月選択時の自動入力が優先されない）
- [ ] React への値セットにネイティブセッター経由のイベント発火を使っている
- [ ] SPA 遷移後もセレクトが再注入される
